Flex basis demo

Intro

This demo visually shows the contributions of flex-grow and flex-shrink to the size of a flex item for different initial sizes flex-basis.

The grey area is the initial size due to flex-basis. The green area is the additional size due to flex-grow. The blue area is the min-content size toward which flex-shrink reduces the size. Note that we can't visualise the for reference since we can't directly visualise the "subtracted size" due to flex-shrink unlike the additional size due to flex-grow.

Instructions:

  1. Toggle different values for flex-basis with deactivated growing / shrinking. Note that 0 behaves the same as min-content due to the lower bound of the minimum size min-content (default).
  2. Resize the browser window to make the boxes in their initial size fill the whole width.
  3. Enable growing and / or shrinking.
  4. Resize the browser window to see the effect.
  5. Observe how for growing the value 0 for flex-basis behaves different from min-content.
  6. Observe how for shrinking it behaves like the value 0 for flex-basis for growing.

Remember, flex-shrink has no effect if flex-basis is smaller than or equal to the minimum size min-content (default). In other words, it makes sense to enable it only for flex-basis: 15rem.

Note: Check browser compatibility first for flex-grow: min-content value on caniuse.
